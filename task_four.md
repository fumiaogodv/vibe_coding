# 需求文档：周期性任务打卡与日记生成器 (Web App)

## 1. 项目概述
需要使用 Python (Flask框架) 开发一个轻量级的个人任务周期管理与打卡 Web 应用。该应用的核心逻辑是基于“起始日”和“循环天数”自动计算当天需要执行的任务，允许用户打卡、记录感想，并一键生成 Markdown 格式的日志以便复制到个人日记中。

项目无需复杂的数据库，推荐使用 SQLite 或 JSON 文件进行本地数据持久化存储，方便后续进行 Docker 容器化部署。

## 2. 核心功能模块

### 模块 A：任务配置页 (Task Management)
- **功能描述**：用于定义和管理所有的周期性任务。
- **交互元素**：
  - **任务输入表单**：包含“任务名称”（String）和“周期天数”（Integer）。
  - **默认逻辑**：添加新任务时，后台自动将其“起始计日（start_date）”设为当天。每日任务的周期天数为 1。
  - **任务列表视图**：展示当前已配置的所有任务及其周期，支持简单的删除功能。

### 模块 B：今日打卡页 (Daily Execution)
- **功能描述**：应用的默认主页，仅展示计算后得出“今天需要做”的任务。
- **逻辑算法**：遍历所有任务，当 `(当前日期 - start_date) % 周期天数 == 0` 时，判定该任务今日需执行。
- **交互元素**：
  - **今日任务列表**：每个任务前有一个 Checkbox（复选框）。
  - **动态输入框**：当勾选某个任务时，可选择性展开一个简短的文本输入框，用于填写具体执行细节（如“哑铃飞鸟4组”）。
  - **全局随笔区**：页面底部提供一个 Textarea（多行文本框），用于输入全天的感受、三餐记录或系统总结。
  - **生成按钮**：一个“生成 Markdown”的提交按钮。

### 模块 C：Markdown 导出模块 (Export)
- **功能描述**：点击“生成按钮”后，将前端页面收集到的打卡状态和文本内容，拼接成格式化的 Markdown 文本。
- **交互元素**：
  - 弹出 Modal（模态框）或在页面侧边栏显示生成的纯文本 Markdown 源码。
  - 提供一个“一键复制”按钮（利用 Clipboard API）。
